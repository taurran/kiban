# Naming Guide

Step-by-step process for naming a new project or product. Usable by a human
walking through it manually or by an AI agent running each phase.

This is always the first step of marketing for any commercially viable project.

---

## Phase 1: Meaning-First Ideation

Start with the CONCEPT, not the name. What does the project DO? What is its
core metaphor? What feeling should the name evoke?

- [ ] Write a 1-sentence description of the project's purpose
- [ ] List 3-5 concept words (e.g., "foundation," "infrastructure," "scaffold")
- [ ] Explore translations in other languages (Japanese, Latin, Greek, etc.)
- [ ] Look for short, memorable words (ideally 2-3 syllables)
- [ ] Generate 5-10 candidate names

**Tips:**
- Meaning-first names age better than clever wordplay
- Foreign language words work well if they are pronounceable in English
- Avoid names that require explanation ("What does that mean?" is a red flag for products; acceptable for frameworks)

---

## Phase 2: Cross-Language and Cultural Check

Before falling in love with a name, verify it does not have negative or
embarrassing meanings in other languages or cultures.

- [ ] Search the word in Google Translate across major languages (Spanish, Chinese, Japanese, Arabic, Hindi, French, German)
- [ ] Check Urban Dictionary for slang meanings
- [ ] Search Reddit and Twitter/X for existing usage of the word
- [ ] Ask a native speaker if the word is suitable (for non-English names)
- [ ] Verify pronunciation is intuitive for English speakers

**Red flags:**
- Word means something vulgar or offensive in another language
- Word is already strongly associated with an existing product or brand
- Pronunciation is ambiguous (multiple common ways to say it)

---

## Phase 3: Availability Sweep

Check that the name is not already taken in the channels you plan to use.

### Required Checks

- [ ] **App Store search** -- search Apple App Store for exact and similar names
- [ ] **Google Play Store search** -- search Play Store for exact and similar names
- [ ] **Domain availability** -- check .com, .app, .ai, .io (use namecheap.com or similar)
- [ ] **GitHub** -- check github.com/{name} org and github.com/{your-username}/{name} repo

### Recommended (If Going Commercial)

- [ ] **USPTO TESS trademark search** -- search https://tmsearch.uspto.gov for conflicts in your trademark class
- [ ] **International trademark** -- check WIPO Global Brand Database if targeting international markets

**Note:** You do not need ALL domains or ALL clean trademark results. You need enough
clearance for your planned distribution channels. A personal framework does not need
a trademark search; an App Store product does.

---

## Phase 4: Connotation Check

The final gut check. Does the name feel right?

- [ ] Say it out loud 10 times -- does it still sound good?
- [ ] Write it in a sentence: "I'm building [name]" -- does it feel natural?
- [ ] Imagine it on an App Store listing -- does it look professional?
- [ ] Ask 2-3 people who have never heard of the project what the name makes them think of
- [ ] Check: does the name scale? Will it still fit if the product grows beyond its initial scope?

---

**Result:** kiban has positive connotations in Japanese -- it is used in engineering
and business contexts (IT kiban = IT infrastructure). No negative meanings found in
other major languages. Pronunciation is intuitive: "kee-bahn."

### Phase 3: Availability Sweep

**Checks performed:**
- GitHub: check your preferred handle for availability
- Domain: kiban.app, kiban.ai -- checked availability
- App Store: "kiban" -- no conflicts (framework is not distributed via App Store)

**Trademark:** Skipped -- kiban is a personal framework, not a commercial product.

### Phase 4: Connotation Check

**Result:** Name feels right. "I'm building kiban" sounds natural. The word scales --
"kiban" as a foundation layer works whether it supports 3 tools or 30. Previous name
(clo-ai-framework) was descriptive but not memorable; kiban is both meaningful and distinct.
